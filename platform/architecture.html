02

The two learning loops

Offline — learn from history

Real settled markets, instant honest grading. dataset loading…

  • 1Dataset — settled KXWC markets + pre-match prices (T-24h/T-4h candlesticks), labeled by result. Split by match — no leakage.
  • 2Trends — calibration by price bucket, momentum, category & liquidity edges → plain-English rules.
  • 3Research loop — try rule injection / retrain on settled-outcome reward; score each on the held-out test set.
  • 4Keep iff better — a change ships only when the target metric beats the champion. Audit: research_log.json

Online — learn from our own bets

Slow but grounded in what the agent actually did.

  • 1Bet — the pipeline above places a real demo bet.
  • 2Settle — browser reads the portfolio; wins/losses land in SQLite.
  • 3Reflect — Holo mines the outcomes for lessons; strategy rules update.
  • 4Retrain — resolved bets become GRPO trajectories with realized P&L reward.
03

What the model optimizes

Risk-adjusted profit from calibrated, well-sized bets

Reward = P&L per $ staked + calibration (confident-and-wrong −0.40) + Kelly-style sizing bonus + liquidity bonus. GRPO samples 4 decisions per market state and shifts probability toward those that beat the group average — no critic network. The bar to clear: the market price itself. A predictor only has edge if it beats the implied-price baseline below.
